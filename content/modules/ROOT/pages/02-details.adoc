== Demo personas

This demo uses two SSO (Keycloak) personas. Both use the common password `{password}`. Use these credentials for the OpenShift console, GitLab, Dev Spaces, and Developer Hub.

* **`{dev_user}`** — Developer persona. Use this when demonstrating the developer experience: DevSpaces, writing code, pushing commits, viewing pipeline results.
* **`{pe_user}`** — Platform engineer persona. Use this when demonstrating platform configuration: DevSpaces operator, Service Mesh, production deployments.

**Tip:** To avoid constant sign-out/sign-in noise during the demo, keep each persona logged in using a separate browser. For example, use Chrome for the developer persona and Firefox for the platform engineer persona (or use an incognito/private window for the second persona). This lets you switch between personas instantly by switching browser windows.

Other services (SonarQube, Argo CD, Vault) require their own admin credentials — see the access details below.

== Demo application

The demo application is the **Parasol Insurance** web application, a Quarkus-based microservices application that handles policy management, claims processing, and customer interactions. Parasol migrated this application from a legacy Java EE platform to Quarkus on OpenShift using the Migration Toolkit for Applications (MTA).

The application has two environments:

* **Dev**: `{parasol_dev_ns}` namespace — parasol-insurance-{parasol_dev_ns}.{subdomain}
* **Prod**: `{parasol_prod_ns}` namespace — parasol-insurance-{parasol_prod_ns}.{subdomain}

NOTE: The application pods will initially be in **ImagePullBackOff** status because the container images have not been built yet. The images are built during the demo when the CI/CD pipeline runs for the first time.

The Tekton pipeline and build resources are in the `{parasol_build_ns}` namespace.

In Section 2, the application and all its components are registered in the Red Hat Developer Hub catalog as a "system" with linked components (frontend, backend, Kafka, database).

In Section 3, the application is extended with an AI-enhanced component that consumes the LLM endpoint and processes business data from Kafka.

== Pre-demo checklist

=== Section 1

. Log into the OpenShift console at {console_url}[{console_url}^] as `{dev_user}` / `{password}`
. Navigate to **Workloads → Topology** and select the `{parasol_dev_ns}` namespace. Confirm you see the Parasol application topology (pods will be in ImagePullBackOff initially — this is expected)
. Open Dev Spaces at {devspaces_url}[{devspaces_url}^] and log in as `{dev_user}` / `{password}` — confirm the dashboard loads
. Verify the AI code assistant works: in **Workloads → Topology** for the `{parasol_dev_ns}` namespace, click the "Edit source code" icon on the Parasol application blue donut. Once DevSpaces opens, wait for the Continue plugin to initialize (this may take a few minutes on first launch)
. Open SonarQube at {sonarqube_url}[{sonarqube_url}^] and log in as `admin` (retrieve password from `sonarqube-admin` secret in the `sonarqube` namespace)
. Open the rhdh-gitops Argo CD at {argocd_url}[{argocd_url}^] and log in as `admin` (retrieve password from `rhdh-gitops-cluster` secret in the `rhdh-gitops` namespace)
. Verify the Tekton pipeline is visible in the OpenShift console: navigate to **Pipelines** in the `{parasol_build_ns}` namespace
. Log into the OpenShift console as `{pe_user}` / `{password}` and open **Service Mesh → Overview** to confirm it loads, then check **Service Mesh → Traffic Graph** to confirm the traffic graph renders

=== Section 2 (in addition to Section 1)

. Open Red Hat Developer Hub at {rhdh_url}[{rhdh_url}^] and log in as `{dev_user}` / `{password}` — confirm the catalog loads
. Verify the Parasol system and its components are registered in the catalog
. Confirm software templates are available in the "Create" section
. Open TPA at {tpa_url}[{tpa_url}^] and confirm SBOM data is visible
. Verify Vault is accessible at {vault_url}[{vault_url}^]:
.. Retrieve the root token: `oc get secret vault-token -n vault -o jsonpath='\{.data.root_token}' | base64 -d`
.. Log in with the root token and verify Parasol secrets are configured at {vault_secrets_url}[{vault_secrets_url}^]
. Open GitLab at {gitlab_url}[{gitlab_url}^] and log in as `{dev_user}` / `{password}` — confirm repository access

=== Section 3 (in addition to Sections 1-2)

. Verify the LLM endpoint is serving and accessible at {llm_endpoint_url}[{llm_endpoint_url}^]
. Confirm the Kafka topic with business data is available and producing messages
. Verify the AI component template is available in RHDH
